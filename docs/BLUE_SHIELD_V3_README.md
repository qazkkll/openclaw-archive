## 一、起源与演进

### 1. 蓝盾v1 — SP500 ML方向预测（废弃）

**设计：** 用随机森林/XGBoost对S&P 500大盘股做方向预测（涨/跌/中性），特征包含MACD/RSI/均线/成交量等，训练集10年。

**死因：** 验证集准确率**49%**，和抛硬币没区别。大盘股走势预测的难度远大于小盘股——大盘股更受宏观事件/财报/利率预期驱动，纯技术面ML模型无法捕捉基本面beta。

**分析来源：** 早期回测日志显示v1模型在2018-2019的验证集上方向准确率始终在47%-52%之间震荡，没有任何正的夏普比率。

### 2. 蓝盾2.0 — 纯评分公式，买入线≥60（亏46%，废弃）

**设计：** 从V5三模型的主力S部分提取评分公式，110分制、6维度加权。候选池50只S&P 500核心大盘股，买入线≥60。

**死亡回测：** 2026-06-11实盘回溯42天（4/1→5/13），评分≥60触发35笔交易，总收益**-46%**。原因是门槛太低（50只候选池中约30-40只≥60），买入了太多弱势股，低胜率+小盈利无法覆盖亏损。

**关键教训：** 蓝盾2.0在一个50只强势股池里挑出30只，甄别度极低——基本上"池子里的就买"。

**分析来源：** `scripts/tst_ld3_backtest.py` 在2026-06-11运行的42天回测报告，输出于 `D:/openclaw/data/`，35个信号的总收益-46%。

### 3. 蓝盾3.0 — 严格入场条件，回测+39.6%（当前活跃）

**设计变更：**
- 买入门槛从≥60提高到**≥80**（保持买入甄别度）
- 退出线从<60提高到**<70**（利润锁定更快）
- 增加**-15%固定止损**
- 持仓上限**10只**

**短期回测（4/1→6/10，70天，50只候选池）：**

| 指标 | 数值 |
|:---|---:|
| 总收益 | **+39.6%** |
| 最大回撤 | **~4%** |
| 胜率 | 48.4% |
| 买入标的 | ARM +183%、MU +93% |
| 止损触发 | 1次（SHOP -15%） |

ARM和MU两只超级强势股主导了收益，但47%的胜率搭配~4%回撤说明"少量高盈亏比+快速止损"的结构有效。

**分析来源：** `scripts/tst_ld3_backtest.py` 回测输出。注意回测期短（70天）、候选池小（50只），样本有限。

### 4. 后续演进：SP500全量扩散（2026-06-12/13）

2026-06-12起，候选池从50只S&P 500核心股**扩至全量S&P 500 503只+扩展池**（HPK、MRDN）。

**直接影响：** 评分≥80的股票从原来的5-8只（10-16%）增加至**75只（15%）**。甄别度大幅下降。

这一变化已触发考虑将买入门槛从≥80上移到≥85甚至≥88，但尚未决策。

## 二、模型设计

### 1. 评分公式架构（满分110分）

蓝盾3.0使用**纯加权评分公式**，不依赖ML模型。数据够（≥60根K线）就能跑，无状态/无训练/无校准。

#### 维度1：趋势排列（30分）

**逻辑：** 均线多头排列是趋势确认的最强信号。价格在均线之上=趋势健康。

| 条件 | 得分 | 说明 |
|:---|---:|:---|
| MA5 > MA20 | +6 | 短期均线上穿中期 |
| MA20 > MA60 | +8 | 中期趋势确认 |
| MA60 > MA120 | +10 | 长期结构向上 |
| 价格 > MA20 | +3 | 价格在中期均线上方 |
| 价格 > MA60 | +3 | 价格在长期均线上方 |
| 三级全排列奖励 | +5（上限30） | MA5>MA20+MA20>MA60+MA60>MA120同时满足 |

**核心逻辑：** 三层排列全对（5>20 AND 20>60 AND 60>120）是"完美多头"信号，此时趋势强度得分可达30分。

#### 维度2：动量持续性（25分）

**逻辑：** 20日/60日正动量表示趋势延续能力强。加入30日过热衰减，防止追高。

| 条件 | 得分 | 说明 |
|:---|---:|:---|
| 基础分 | +10 | 每个股票都有基础动量分 |
| 20日收益 > 3% | +5 | 中期动量确认 |
| 20日收益 > 10% | +5 | 中期强力动量 |
| 60日收益 > 5% | +3 | 长期动量确认 |
| 60日收益 > 15% | +3 | 长期强力动量 |
| 60日收益 -5%~5% | **-3** | 横盘，无动量 |
| 30日收益 > 40% | **扣分** | 过热保护，每超5%扣1分，最多扣10分 |

**核心逻辑：** 默认10分基础分，正动量逐步累加至25。但短期过热（30日涨超40%）会启动衰减扣分——这个逻辑在2026年ARM暴涨中有效。

#### 维度3：MACD能量（25分）

**逻辑：** MACD金叉+柱线扩大是最强短期买入信号（原V5-S核心）。

| 条件 | 得分 | 说明 |
|:---|---:|:---|
| MACD线 > 信号线 | +8 | MACD在零轴上方的趋势信号 |
| 柱线上穿零轴（金叉） | +10 | 最强买入信号 |
| 柱线正且持续扩大 | +7 | 加速上涨 |
| 柱线正但缩小 | +4 | 动能减弱但正区 |
| 柱线高空回落（MACD顶背离） | **-5** | 趋势逆转信号 |

**核心逻辑：** 金叉/上穿零轴给最高的10分，因为这是趋势确认的"起点"。后续扩大/缩小反映强度持续度。

#### 维度4：均线偏离度（10分）

**逻辑：** 价格偏离MA20的百分比——太近（<1%）说明没突破，太远（>15%）说明追高风险大。

| 条件 | 得分 | 说明 |
|:---|---:|:---|
| 偏离MA20 1%-8% | +10 | **甜点区间** — 已确认突破且不追高 |
| 偏离MA20 -2%~1% | +5 | 接近突破，接近MA20 |
| 偏离MA20 8%-15% | +6 | 仍安全但偏热 |
| 偏离MA20 > 15% | +3 | 追高风险大 |
| 偏离MA20 < -5% | +0 | 弱于均线，不参与 |

**核心逻辑：** 最关键的甜点区间是价格在MA20上方1%-8%，既有突破确认又有空间。这是大盘股评分中筛选"刚启动而非追高"的核心维度。

#### 维度5：RSI位置（10分）

**逻辑：** RSI 50-65是主升段的"甜点"——不超买、不弱势。

| 条件 | 得分 | 说明 |
|:---|---:|:---|
| RSI 50-65 | +10 | **主升段最佳区域** |
| RSI 35-50 | +7 | 刚脱离超卖，可能启动 |
| RSI 65-75 | +5 | 偏热但可控 |
| RSI < 30 | +4 | 超卖区可能反弹 |
| RSI > 80 | +2 | 超买，扣分 |

**核心逻辑：** RSI在50-65是最佳持仓区域——既不是弱势（<50）也不是超买（>75）。大于80的极端超买直接给2分。

#### 维度6：52周位置（10分）

**逻辑：** 价格在52周区间中位区域（30-70%）反弹空间最大，接近高位（>85%）风险大。

| 条件 | 得分 | 说明 |
|:---|---:|:---|
| 52周百分位 30-70% | +10 | **中位区域，空间最大** |
| 52周百分位 70-85% | +6 | 偏高但可控 |
| 52周百分位 15-30% | +7 | 低位，可能反转 |
| 52周百分位 < 15% | +4 | 极低位，趋势不强 |
| 52周百分位 > 85% | +2 | 高位，风险大 |

**核心逻辑：** 大盘股偏离52周高点太远（>85%）之后的空间有限。中位区域（30-70%）给满分，因为向上空间最大。

### 2. 总分计算

```
总分 = 趋势排列(30) + 动量持续性(25) + MACD能量(25) + 均线偏离(10) + RSI位置(10) + 52周位置(10)
     = 110 分（封顶）
```

**信号线：**

| 评分区间 | 信号 | 动作 |
|:---|---:|:---|
| ≥80 | 🟢💪 **强势买入** | 趋势+动量+MACD三联共振，值得持仓 |
| 70-79 | 🟡 **观望/监视** | 接近门槛但不达标 |
| <70 | 🔴 **退出/规避** | 评分弱，不应持仓 |
| — | 🛑 **止损 -15%** | 任何持仓亏损达15%即出，与评分无关 |

### 3. 为什么是6维度而不是V1的BP/ATR等？

早期v1试验了布林带位置（BP, 15分）和ADX趋势（15分），但回测中发现：
- **布林带位置**对大盘股区分度不高——S&P 500中大市值公司价格大部分时间在上中轨之间
- **ADX趋势**在大盘股中取值偏大（趋势性强），区分度下降

蓝盾3.0的6维度（趋势排列+动量+MACD+均线偏离+RSI+52周位）在最终回测中提供了最佳的区分度。

**分析来源：** `scripts/tst_ld3_backtest.py` 中各维度消融测试。回测比较了6维 vs 8维（含BP/ADX）的CAGR差距<2%，选择更简洁的6维。

## 三、数据源

### 1. 数据管线

| 数据 | 来源 | 格式 | 更新方式 | 频率 |
|:---|:---|:---|:---|:---:|
| 日K线（OHLC） | yfinance `yf.download` 批量 | pandas DataFrame | 实时拉取60-250天 | 每跑一次拉一次 |
| SP500列表 | SP500符号JSON | JSON (`sp500_symbols.json`) | 手动更新 | 季度更新 |
| 扩展池 | 硬编码在脚本中 | 列表 (`EXTRA_POOL`) | 手动维护 | 有新增持仓时更新 |

**不依赖任何本地parquet/h5数据文件。** 每次运行直接从yfinance拉取。

### 2. yfinance使用细节

- 调用方式：`yf.download(tickers, period='6mo', group_by='ticker', threads=True)` 批量拉取2-6个月
- 速度：505只在约**55秒内**完成（含3只失败重试，实际成功502只）
- 偶发失败：约3/505只会报错（HTTP 429/超时），自动记为0分
- **不缓存数据，不维护本地K线** — 每次运行都是实时拉取

### 3. 候选池来源

- SP500列表：从 `D:/openclaw/data/sp500_symbols.json` 加载
- EXTRA_POOL：硬编码 `['HPK', 'MRDN']`（可能是个人持仓）
- 池大小：503 + 2 = 505只

### 4. 数据时效性

- yfinance返回的是截至最近交易日的日K线数据
- **盘中可跑**（拿到的是截至上一交易日的收盘价）
- **盘后更准确**（最新收盘价已被更新）
- 不保证yfinance收盘价与实务券商数据完全一致（一般差异<0.1%）

## 六、与绿箭V8-Lottery的关系

### 双模型架构

```
    🟢 绿箭 V8-Lottery                  🛡️ 蓝盾 3.0
    ────────────────────────             ────────────────────
    $1-10 彩票爆发预测                   大盘技术评分
    ~600只小盘股                         S&P 500 503只 + 扩展
    XGBoost ML 43特征 → 概率             纯公式评分 6维度 → 110分制
    预测5日涨>50%极端爆发                 择时择股（趋势+动量）
    买入: top5评分>阈值                   买入: 评分≥80（最多10只）
    top5命中率 17.1%                      持仓上限10只
    命中票平均 +90%                      止损 -15%
```

### 使用场景

| 场景 | 用的模型 |
|:---|---|
| 用户说"绿箭" | 跑V8-Lottery（$1-10小盘） |
| 用户说"蓝盾" | 跑蓝盾3.0（S&P 500大盘） |
| 用户说"推荐" | 跑融合推荐（双模型同时） |
| 用户说"持仓分析" | 双模型分别评分 |

### ⭐ 共振信号

绿箭V8选出的小盘爆发候选 + 蓝盾3.0选出的趋势强股极少重叠（因为候选池不同），**但当某只股票同时被两个模型看好时，置信度极高。**

共振信号的典型场景：
- 某只$5-10的小盘股被V8高度看好（预测5日涨>50%）
- 该股同时在蓝盾3.0的S&P 500候选池中（或刚好是大盘蓝筹股）
- 此时**极度可信**，应重仓

共振范围有限——V8跑的是$1-10小盘股，蓝盾3.0跑的是S&P 500，重叠面很小。

### 并行运行顺序

```
绿箭V8 (4-5小时) → 蓝盾3.0 (55秒) → 融合推荐 (1秒)
```

绿箭V8是瓶颈（下载全小盘+43特征推理），蓝盾3.0快速同步后即可融合。

## 七、缺点与局限性

> ⚠️ 每条局限性都附有**分析来源**，供后续复现或改进。

### 1. ⚠️ 候选池扩散到505只后，评分≥80的甄别度大幅下降

**现象：** 2026-06-13全量评分中，502只成功评分的股票有**75只评分≥80**（15%），这意味着买入信号太普遍——在505只里选出75只，相当于"买了池子里最强的15%"，而不是真正精选。

**分析来源：** `D:/openclaw/data/ld3_scored_2026-06-13.json` 中的得分分布。对比旧50只候选池≥80分的比例（5-8只，10-16%），总量几乎没变（都是10-16%）——但白噪声增加了10倍。

**影响：**
- 以前50只里看Top5-8即可，现在要看75只
- 机器学习无法帮助进一步过滤（蓝盾是纯公式）
- 需手动观察评分在85+的更精选层（6/13只有9只评分≥90）
- **购入门槛可能需要调整到≥85或≥88**

### 2. ⚠️ 纯技术面，无基本面/事件驱动

**现象：** 蓝盾3.0只根据价格和成交量数据评分。财报利好/利空、新闻事件、宏观利率变动、评级调整完全不在模型中。S&P 500大盘股受这些因素影响极大（尤其是NVDA/AAPL/MSFT等权重股）。

**分析来源：** `v5s_calc()` 和 `v5s_score()` 的代码——输入只有close/high/low三部曲，没有news/FCF/PE/insider交易等基本面维度。

**影响：**
- 财报周前一天可能给出高分但财报后跳空下跌
- 利率决议日所有股票同步波动，评分失效
- 评级下调导致单日暴跌，止损被轻松触发

### 3. ⚠️ 止损失误案例（SHOP -15%后反弹）

**现象：** 2026年4-5月回测中，蓝盾3.0在SHOP上触发了-15%止损，但SHOP在止损后迅速反弹+20%+。

**分析来源：** `scripts/tst_ld3_backtest.py` 回测记录：SHOP在蓝盾评分系统买入后，先跌-15%触发止损，随后5个交易日反弹+23%。**止损保护了本金，但错过了后续涨幅。**

**影响：**
- -15%止损对于波动率>50的个股（如SHOP）来说太紧
- 但调宽止损（如-25%）又会拖累回撤控制
- **当前方案：-15% + 用波动率感知止损？不，3.0版本暂不实施**
- 长期方案：可以考虑引入ATR倍数止损（如2×ATR）

### 4. ⚠️ 不能处理盘前/盘后跳空

**现象：** 蓝盾3.0基于日K线收盘价评分。盘前大幅高开/低开后的开盘价与昨日收盘价之间出现跳空，评分不反映盘前动态。

**分析来源：** `us_ld3_daily_score.py` 调用 `yf.download` 获取的只有日K线（open/high/low/close/volume），没有盘前盘后数据。中国用户在美股开盘时使用评分的场景下，跳空情况很常见。

**影响：**
- 盘前已涨3%的开盘价能让评分在当天失效
- 盘前大跌的恐慌抛售无法被评分提前识别
- **要解决需要minishare rt_us_k（盘中实时价）做盘中重评分**

### 5. ⚠️ yfinance数据的质量风险

**现象：** 每次运行从yfinance拉取数据，约3/505只会因为HTTP 429或超时而评分失败。2026-06-13的505只中有3只评分失败（502只成功）。

**分析来源：** `us_ld3_daily_score.py` 运行日志中的 `yf.download` 报错信息。

**影响：**
- 失败率~0.6%，可以忽略，但涉及某只持仓股时可能误判
- yfinance在美股开盘后有时延（数据更新慢于实际）
- 偶发数据缺失（split/adjustment导致的历史数据断裂）

### 6. ⚠️ 评分≥80在旧50只池中效果最好

**现象：** 蓝盾3.0的回测全部基于旧50只候选池。扩散到505只后，75只达标股票中大量是之前从未出现在候选池中的"边缘S&P 500"股票（如ROST/IBKR/TER等），其回测表现未经验证。

**分析来源：** `scripts/tst_ld3_backtest.py` 只使用旧50只池回测。扩散后的实际表现尚无数据支撑。

**影响：**
- 蓝盾3.0扩散后的真实胜率和收益**未经回测验证**
- 边缘S&P 500股票的流动性/波动率与核心大盘不同
- 建议：先在505只池上跑1-2个月积累实盘数据后重新评估

## 八、风险边界

### 已知风险矩阵

| 风险 | 概率 | 影响 | 应对 |
|:---|---:|:---:|:---|
| 评分≥80在505只池中甄别度不足 | 🔴 高 | 🟠 中 | 考虑上调买入门槛至≥85或≥88 |
| 50只池回测无法代表505只池实盘表现 | 🔴 高 | 🔴 高 | 等待1-2个月实盘数据后评估 |
| yfinance数据延迟/失败 | 🟢 低 | 🟡 中 | 自动重试+失败记录为0分 |
| 大盘股跳空下跌 | 🟡 中 | 🟡 中 | 止损保护，但需响应 |
| 纯技术面错过财报/事件 | 🟡 中 | 🟡 中 | 人工review新闻 |
| 止损过早（如SHOP反弹） | 🟡 中 | 🟢 低 | 保护本金优先，不要因懊恼修改止损 |
| 绿箭V8与蓝盾3.0同步发出相反信号 | 🟢 低 | 🟢 低 | 以蓝盾为准（大盘） |
| 小盘股误入蓝盾池（市值小于200亿） | 🟢 低 | 🟢 低 | S&P 500自动过滤小盘 |

### 关键假设

1. **大盘股技术面趋势具有延续性** — 蓝盾3.0的核心假设
2. **yfinance日K线为准** — 不处理盘中跳空
3. **-15%止损保护本金优先** — 宁可错过反弹也不能大幅回撤
4. **S&P 500全量池中评分≥80的股票会稳定出现** — 6/13验证：75只，足够

### 观察期建议

| 指标 | 建议观察期 | 如果偏差则 |
|:---|---:|:---|
| 评分≥80命中率 | 1-2个月实盘 | 上调门槛到≥85 |
| 大盘股胜率 | 3个月 | 重新评估扩散合理性 |
| 止损触发率 | 持续观察 | 如果>30%暂缓新开仓 |
