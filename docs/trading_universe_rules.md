# 实战数据池规则 (Trading Universe)
# 最后更新: 2026-06-26
# 原则: 训练池宽（数据多=模型准），实战池严（信号干净=可交易）

---

## 🏹 绿箭V12 — $1-$10宇宙

### 训练池（模型训练用，不做过滤）
- 价格: $1.00 ≤ close ≤ $10.00
- 无流动性要求
- 无符号过滤
- 目的: 最大化训练数据量（~2100只）

### 实战池（每日评分用，严格过滤）
- 价格: $1.00 ≤ close ≤ $10.00
- 排除权证: sym以W结尾
- 排除SPAC单位: sym以U结尾
- 排除权利: sym以R结尾
- 排除杠杆/反向ETF: 黑名单130+只（见universe_filter.py）
- 流动性分层:
  - $1.00-$2.00: 日均美元成交 ≥ $200万/日（低价股必须高流动性）
  - $2.00+: 日均美元成交 ≥ $50万/日
- 过滤后: ~1000-1100只

### 低风险标记
- $1.00-$1.50: ⚠️ 退市风险区，仅限流动性极好的标的
- $1.50-$2.00: 关注区
- $2.00+: 正常区

## 🛡️ 蓝盾V10 — >$10宇宙

### 训练池（模型训练用）
- 价格: close ≥ $10.00
- 无流动性要求
- 目的: 最大化训练数据量（~10000只）

### 实战池（每日评分用，严格过滤）
- 价格: close ≥ $10.00
- 排除权证: sym以W结尾
- 排除SPAC单位: sym以U结尾
- 排除权利: sym以R结尾
- 排除杠杆/反向ETF: 黑名单130+只 + 名称模式匹配
- 流动性: 日均美元成交 ≥ $500万/日
- 过滤后: ~3000-3500只

### 异常值处理
- score > 10倍中位数: 标记为⚠️ outlier，手动核查
- 原因: 某些特征（RSI/MACD极端值）会导致个别标的分数异常高

## 实现文件
- 过滤器: `scripts/us/universe_filter.py`

- 每日自动执行: 评分脚本内置过滤，无需额外步骤
