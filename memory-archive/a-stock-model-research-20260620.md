# A股模型研究：历代复盘 + 行业前沿 + 升级方案

## 一、项目内A股模型复盘

### A1（资金流模型）— ❌ 失败
- **设计**：三层架构（L1市场状态 + L2数据 + L3评分 + L4决策）
- **死因**：纯资金流选股 CAGR **-26.94%**（2017-2026回测）
- **教训**：A股资金流（北向/大户/散户）存在严重滞后性——暴涨时大单流出、暴跌时大单流入。资金流降级为"实时盘中提示"，不做独立选股

### A2/L3（纯L3回归评分）— ⚠️ 可用但需改进
- **设计**：XGBoost回归，38特征（18技术+20资金流），预测10日涨幅
- **Walk-Forward 7段验证**：平均CAGR +33.14%，夏普1.03
- **问题**：
  1. 2023年亏损-12%（窄幅震荡+AI主题行情导致轮动失序）
  2. 评分偏高未校准（输出+35%预期收益实际不可能）
  3. 最大回撤-37.51%（需-15%止损）
  4. 绝对值不可信，仅用于排序（这是对的！）

### V3（蓝盾3.0 美股）— ❌ 失败
- **设计**：纯公式评分，110分制，6维度加权
- **死因**：v1 ML方向预测49%准确率（和抛硬币没区别），v2公式≥60买入亏46%
- **教训**：大盘股走势受宏观事件/财报/利率驱动，纯技术面ML无法捕捉基本面beta

### V4（蓝盾V4 美股）— ✅ 成功
- **设计**：XGBoost二分类，19特征，AUC 0.7152
- **突破**：截面排名+分类优于回归，成为后续V6/V11的基础

## 二、行业前沿研究

### 1. OW-XGBoost（学术论文 2024）
**来源**：PeerJ Computer Science，对外经贸大学
**核心创新**：融合标签（Fused Labels）— 不是预测单一收益率，而是把5个指标融合成一个标签：
- 收益：累计收益、超额收益(alpha)
- 风险：波动率、最大回撤
- 风险调整：夏普比率
- 最优权重：[0.4, 0.4, 0.1, 0.1, 0]

**结果**：
| 指标 | OW-XGBoost | 单标签XGBoost |
|------|-----------|--------------|
| 收益 | 30.09% | 10.71% |
| 超额 | 22.85% | 4.55% |
| 夏普 | **3.113** | 0.853 |
| 最大回撤 | 5.9% | 8.68% |
| Beta | 0.339 | 0.458 |

**关键发现**：月度调仓 > 周度调仓，大盘股 > 小盘股，短期训练窗口(2月) > 长期(6月)

### 2. AShare-AI-Stock-Picker（GitHub开源）
**地址**：github.com/stlin256/AShare-AI-Stock-Picker
**技术栈**：LightGBM + Optuna超参优化 + akshare数据源
**特点**：
- 二分类（5日后涨/跌）
- RSI/MACD/CCI/ATR/ADX技术指标
- 推荐评级：A级(≥0.75) / B级(0.6-0.75) / C级(<0.6)
- 训练时间约5小时

### 3. QuantML Factor Zoo（GitHub 1049因子）
**地址**：github.com/QuantMLResearch/QuantML
**核心价值**：
- **1049个Alpha因子**：量价因子详细分类（振幅/标准差/高阶矩/成交占比/流动性/量价相关）
- **Qlib集成**：微软开源量化平台，支持截面排名
- **DeepSeek因子挖掘**：用LLM自动生成Alpha因子公式
- **MASTER模型**：AAAI 2024论文，市场引导的股票预测Transformer

### 4. 量化67工程指南（工业界最佳实践）
**来源**：quant67.com
**核心观点**：

**标签构造**：
- **截面排名转换**（工业界标准）：先做秩变换再正态化，模型学习"相对位次"
- **Triple-Barrier标签**：定义止盈/止损/时间三条边界，更贴近真实交易
- **Meta-Labeling**：二阶模型预测"一阶信号是否值得跟"

**模型选择**：
| 等价样本量 | 推荐模型 |
|-----------|---------|
| 10⁵-10⁶ | **LightGBM（A股默认选择）** |
| 10⁶-10⁷ | 深LightGBM / 浅神经网络 |
| > 10⁷ | 深度网络 |

**防过拟合**：
- **Purged K-Fold**：训练/测试划分时剔除标签时间重叠的样本
- **Embargo**：测试集后增加隔离带
- **截面标准化**：去极值→截面Z-score→秩变换→反正态

### 5. 深度学习前沿
- **GriffinNet**（国信证券研报）：在A股沪深300/800收益预测中显著优于LSTM/XGBoost
- **Transformer**：AAAI 2024的MASTER模型，市场引导的注意力机制
- **强化学习**：AlphaGen用RL公式化生成Alpha因子
- **LLM因子挖掘**：DeepSeek自动生成Alpha因子公式

## 三、A2模型诊断：为什么2023年亏12%？

**根因分析**：
1. **标签问题**：A2预测绝对10日涨幅，在趋势行情中有效，但在震荡行情中信号噪声大
2. **特征问题**：38个特征中20个是资金流，但A1已证明资金流独立选股CAGR -26.94%。资金流特征可能是噪声而非信号
3. **没有截面标准化**：直接用原始特征值，没有做行业中性化/市值中性化
4. **没有Purged K-Fold**：Walk-Forward虽然用了时间分割，但没有Purge和Embargo

## 四、A股模型升级方案

### 方案A：LightGBM截面排名（推荐，最快见效）

**核心改动**：
1. **标签**：从"预测绝对涨幅"改为"截面排名"（用`lambdarank`目标函数）
2. **特征**：从38个精简到25个核心因子，去掉大部分资金流特征
3. **标准化**：截面Z-score + 行业市值中性化
4. **训练**：Purged K-Fold + Embargo
5. **调仓**：月度调仓（参考OW-XGBoost结论）

**新增特征**（替换资金流）：
- 截面相对强度（RSI的截面百分位）
- 行业动量（行业内排名）
- 市值因子（对数市值）
- 波动率因子（特质波动率）
- 量价背离因子（量缩价涨/量增价跌）

**预期改善**：
- 排名模型 > 回归模型（美股已验证）
- 截面标准化消除行业/市值偏差
- Purged K-Fold防止过拟合
- 月度调仓降低交易成本

### 方案B：OW-XGBoost融合标签（中期，需更多实验）

**核心改动**：
1. **标签**：融合5指标（收益+alpha+波动率+回撤+夏普），权重网格搜索
2. **模型**：XGBoost回归（保持现有架构）
3. **调仓**：月度，最小方差组合优化

**适用场景**：如果方案A排名模型不够好，可以尝试融合标签来平衡收益和风险

### 方案C：LightGBM + LLM因子挖掘（长期探索）

**核心改动**：
1. **因子挖掘**：用DeepSeek/LLM自动生成Alpha因子公式
2. **因子库**：参考QuantML的1049因子库扩展
3. **模型**：LightGBM截面排名（同方案A）

**适用场景**：当前38特征可能不够，需要更多因子来捕捉A股特有的市场微观结构

## 五、推荐实施路径

### 第一阶段：快速验证（1-2天）
1. 在现有A2数据上，把XGBoost回归改为LightGBM `lambdarank`
2. 标签改为截面排名（每日全市场排名）
3. Walk-Forward验证，对比A2的CAGR 33%和夏普1.03

### 第二阶段：特征优化（3-5天）
1. 去掉资金流特征（已有证据证明是噪声）
2. 加入截面特征（行业中性化、市值中性化）
3. 加入量价背离、波动率因子
4. 重新训练+验证

### 第三阶段：生产部署（1周）
1. 集成到现有评分管线
2. 添加Purged K-Fold验证
3. 月度调仓策略
4. 与美股系统统一架构

## 六、关键结论

1. **A2的核心问题是标签和特征，不是模型算法**
   - 预测绝对涨幅 → 改为截面排名
   - 资金流特征占比太大 → 精简或去掉
   
2. **行业共识：LightGBM是A股选股的默认选择**
   - 样本量10⁵-10⁶级别最适合
   - 比XGBoost更快，支持lambdarank目标函数
   
3. **截面标准化是必须的**
   - 行业中性化 + 市值中性化
   - 防止因子被行业/市值伪装
   
4. **防过拟合比算法选择更重要**
   - Purged K-Fold + Embargo
   - 月度调仓 > 周度调仓
   - 短训练窗口(2月) > 长期(6月)
   
5. **美股V6的成功经验可以直接复用**
   - 截面排名 > 回归
   - 宏观特征（类似美股的VIX/SPY）可以考虑加入A股版（沪深300/VIX等价物）
