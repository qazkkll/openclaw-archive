## 一、起源与演进

### 1. 从V7.5到V8-Lottery的困境

**V7.5（旧绿箭）** 是全市场小盘模型（1602只 $3-$20），标签"5日涨幅>5%"，51个特征（含34个股+14全局指数+3基本面），回测夏普6.21。成绩漂亮，但实盘没有回测那么惊艳。

**原因分析：**
- 旧标签（>5%）太温和。对$1-10低价股来说，"涨5%"几乎就是猜方向——随机涨跌的波动都超过5%
- 全市场1602只候选大量质地平庸，拉低特征稳定性
- 用户的核心诉求是**找下一支暴涨股**，不是"温和上涨+5%"
- 基本面特征（PE/PEG/股息率）对低价小盘的预测价值极低，反而是噪音

**分析来源：** `us_v7_5_report.json` 中V7.5模型训练日志 + 2026-05-06 实盘回顾笔记（见 memory/2026-05-06.md）

### 2. 转折点：问题重定义

> 不是"5天内能涨多少"，而是**"5天内有没有可能翻倍"**

这彻底改变了模型架构：

| 维度 | V7.5（旧） | V8-Lottery（新） |
|:---|---:|:---|
| 候选池 | 1602只 $3-$20 | ~600只 **$1-$10** |
| 标签 | fwd_5d_ret > 5% | **fwd_5d_ret > 50%**（极端爆发） |
| 标签发生率 | ~10% | **~0.44%（极度稀缺）** |
| 特征 | 51个（含宏观/基本面） | **43个纯技术+交叉** |
| 目标 | 预测涨跌方向 | **预测尾部爆发** |
| 心态 | 追求命中率 | **追求盈亏比** |

**分析来源：** 2026-06-08 白板讨论结论（记录于 memory/2026-06-08.md）

### 3. 模型进化时间线

```
V7.5 (2026-05) — 全市场51特征，预测>5%，夏普6.21
  │
  ├─ G2 (L50-G2) — 加4个交叉特征 → 更差（倒退）
  │
  ├─ L30 — $1-10适配: 36特征+L30标签 → top5 vs top10 +7.4%改善
  │
  ├─ L50 — L30升级为50%标签 → 跨月验证全面胜出 ← 基础版本
  │
  ├─ L50-Feat — +6个"潜伏期"交叉特征 → val_AUC 0.7452（持平），17.1%命中率无变化
  │
  ├─ L50-Weighted — 标签加权回归 → 14.3%命中率（倒退）
  │
  ├─ 后处理过滤（硬过滤+软过滤）→ 5.7%~8.6%严重倒退
  │
  └─ 超额收益/均轮动检查 → 月度统计无系统性偏倚
       │
       ▼ V8-Lottery（定版，2026-06-12）
```

**核心发现（三个独立方向交叉验证）：**
- **方向1（加特征）：** L50-Feat加6个潜伏期特征，val_AUC 0.7452 vs 原L50 0.7452，无任何提升
- **方向2（改损失函数）：** L50-Weighted回归式，Top5命中率14.3%，比17.1%倒退了
- **方向3（后处理过滤）：** 硬过滤（RSI>40/volume>X）→ 5.7%；软过滤（条件权重）→ 8.6%，全部低于17%
- **结论：在yfinance OHLCV数据下，17.1%是信息提取天花板，三个独立方向全部撞墙**

**分析来源：**

- L50-Feat：`us_v7_5_l50_feat_report.json`（val_AUC=0.7452 → **持平**）

## 二、模型设计

### 1. 模型架构

```
输入（43个特征，36基础 + 7交叉）
  ├── K线特征（24个）：均线/MACD/KDJ/布林带/成交量/DMI/位置
  └── 交叉特征（7个）：价格×成交量组合、ADX×RSI、RSI×KDJ、布林×成交量
        ↓
  XGBoost二分类器：深度8, lr=0.05, subsample=0.8, colsample=0.9, 早停于39棵, scale_pos_weight=228
        ↓
  输出：margin → Platt校准 → score 0-100
        ↓
  候选~600只排序 → Top5推荐
```

### 2. 训练参数

| 参数 | 值 | 说明 |
|:---|---:|:---|
| objective | binary:logistic | 二分类 |
| eval_metric | auc | 验证标准 |
| max_depth | 8 | 树深度 |
| learning_rate | 0.05 | 学习率 |
| subsample | 0.8 | 行采样 |
| colsample_bytree | 0.9 | 列采样 |
| min_child_weight | 2 | 叶子最小权重 |
| gamma | 0.1 | 分裂最小增益 |
| reg_alpha | 0.3 | L1正则 |
| reg_lambda | 2.0 | L2正则 |
| tree_method | gpu_hist | GPU加速 |
| scale_pos_weight | 228.3 | 正负样本权重比 |
| best_iteration | 39 | 早停于第39轮 |
| train_rows | 935,634 | 训练样本 |
| val_rows | 205,764 | 验证样本 |
| pos_rate (正样本率) | 0.44% | 标签极端稀缺 |

**分析来源：** `us_v7_5_l50_report.json` 中 `params` 字段

### 3. 验证成绩

| 集合 | AUC |
|:---|:---:|
| 训练集 | 0.9773 |
| 验证集 | 0.7452 |
| 测试集 | **0.7719** |

**分析来源：** `us_v7_5_l50_report.json` 中 `train_auc/val_auc/test_auc`

> ⚠️ 注意：0.745 val_AUC + 0.44%正样本率 = 模型能学东西但信号很弱。这解释了为什么Top5命中率17.1%（大约只有14%的候选能识别），而Top10命率降到8.6%（随机水平）。

### 4. 43个特征详解

#### 基础K线特征（24个）

| 分类 | 数量 | 特征名 | 说明 |
|:---|:---:|:---|:---|
| 均线类 | 5 | `ma5_ratio, ma20_ratio, ma60_ratio` + `ema12, ema26` | 价格偏离各均线% + 指数均线值 |
| MACD类 | 3 | `macd, macd_signal, macd_hist` | 标准MACD三件套 |
| KDJ类 | 3 | `k, d, j` | 标准KDJ随机指标 |
| 布林带类 | 4 | `bb_upper, bb_lower, bb_width, bb_position` | 布林带上下轨+带宽+位置百分位 |
| 成交量类 | 4 | `vol5, vol20, vol_ratio, vol_ratio_ma20` | 绝对量+相对量+比值 |
| DMI类 | 3 | `adx, plus_di, minus_di` | 方向性运动指数 |
| 位置类 | 2 | `price_position, price_position_60` | 价格在XX日均线位置百分位 |

#### 交叉特征（7个）

| 特征名 | 组合逻辑 | 设计意图 |
|:---|---:|:---|
| `vix_close` | VIX收盘价 | 市场恐慌情绪（全局） |
| `close_log` | log(收盘价) | 价格量纲归一化 |
| `close_x_vol` | 收盘价 × 成交量 | 价格-成交量交互 |
| `plus_di_x_low_vol` | plus_DI × 低成交量 | 趋势启动但缩量的信号 |
| `adx_x_rsi` | ADX × RSI | 强弱+趋势强度组合 |
| `bb_x_vol` | 布林带宽 × 成交量 | 布林挤压放量突破信号 |
| `rsi_x_kdj` | RSI × KDJ交叉 | 多指标共振信号 |
| `low_price` | 是否超低价（<$3） | 统计表明超低价波动更大 |

**特征重要性（Top 15 by Gain）：**

| 排名 | 特征 | Gain | 说明 |
|:---:|:---|:---:|:---|
| 1 | bb_width | **12.4%** | 布林带宽是预测爆发的最强指标 |
| 2 | bb_x_vol | 5.3% | 布林放量=即将选择了方向 |
| 3 | bb_lower | 4.7% | 下轨是否被击穿 |
| 4 | close_log | 3.8% | 对数价格归一化 |
| 5 | bb_upper | 3.4% | 上轨接近程度 |
| 6 | vol5 | 3.2% | 5日均成交量 |
| 7 | minus_di | 3.0% | 负向运动（超卖） |
| 8 | ema26 | 2.9% | 短期趋势 |
| 9 | vol20 | 2.9% | 中期成交量 |
| 10 | d | 2.8% | KDJ-D值 |
| 11 | ma60_ratio | 2.7% | 远离MA60程度 |
| 12 | ema12 | 2.6% | 极短期趋势 |
| 13 | macd_signal | 2.6% | MACD信号线 |
| 14 | vol_ratio_ma20 | 2.5% | 成交相对活跃度 |
| 15 | macd | 2.3% | MACD差值 |

> **总结：** 布林带 + 成交量 = 模型核心感知。Top3特征全是布林带相关（bb_width, bb_x_vol, bb_lower），合计贡献22.4%的重要性。价格的收紧-放量模式是彩票爆发前的最强信号。

**分析来源：** `us_v7_5_l50_report.json` 中 `top15_features`；L50-Feat报告中 `feature_importance_top20`

### 5. 评分校准

原始XGBoost输出是margin（逻辑回归前的分值），不是概率。经过Platt校准转换为0-100评分。

校准逻辑：
```
1. Platt拟合法： score = 100 / (1 + exp(-(4.54 * margin - 3.73)))
2. z-score归一化： z = (margin - mean_shift) / std
3. sigmoid映射0-100： score = 100 / (1 + exp(-z * 1.5))
```

校准后分数（范围0.1-94.7，均值26.9）不是概率，只用于排序比较。

**注意：** 评分≥85才算买信号。生产运行中Top20平均分85.1。

**分析来源：** `DUAL_MODEL_GUIDE.md` "评分说明"章节；`scored_v75_lottery_2026-06-13.json` 中top20_avg_score=85.1

## 五、使用调教

### 1. 最佳用法（一句话）

> **每天跑评分，取Top5等权买入，每只2%仓位，持有5个交易日，止损 -15%。**

### 2. 买入卖出规则

| 动作 | 规则 |
|:---|:---|
| 买入候选 | Top5，评分≥85，当日不在持仓中 |
| 买入价 | **次日开盘价**（当日收盘出信号，次日开盘执行） |
| 仓位 | 等权分配（总现金÷5），单只≤总资金的10% |
| 持有期 | 5个交易日 |
| 止损卖出 | 收盘价 < 买入价 × 0.85（-15%）→ 清仓 |
| 到期卖出 | 持仓天数 ≥ 10个交易日 → 清仓 |
| 再评分 | 每日运行，滑动窗口更新（非滚动每日买卖） |

### 3. 为什么Top5而不是Top3/Top2？

| 方案 | 月回报率 | 命中率 | 结论 |
|:---|---:|:---:|:---|
| **Top5等权** | **+16.5%** | **17.1%** | ✅ **最佳平衡** |
| Top3等权 | +13.4% | 14.3% | ❌ 浪费机会，少7.1%收益 |
| Top2重仓 | +21.7% | 21.4% | ⚠️ 波动大，样本量太小（7轮），风险不可控 |
| Top10等权 | +11.2% | 8.6% | ❌ 命中率大幅下降，稀释信号 |

### 4. 不用评分阈值过滤

三个优化方向全部结论：**越过滤越差。**
- 原始Top5: 17.1% ✓
- 硬过滤（RSI>40/volume>X）: 5.7% ✗
- 软过滤（条件权重）: 8.6% ✗
- 评分阈值筛选: 无提升

## 六、缺点与局限性

> 每条带分析来源（回测日志/生产输出/实验记录）。让后面的人能复现结论。

### 1. 🔴 17.1%命中率天花板（信息瓶颈）

**问题描述：** 在纯yfinance OHLCV数据下，独立三个优化方向全部撞到17.1%命中率天花板。加特征（+6潜伏期交叉）、改损失函数（加权回归）、后处理过滤——无一突破。

**影响范围：** 这是一条硬边界。再优化yfinance数据已经烧了——需要**外部新数据源**（期权链、暗池成交量、社交媒体情绪）才能突破。

**严重程度：** 高 —— 意味着模型性能已封顶，不要期待新版本能显著超越17.1%。

**分析来源：**
- L50-Feat实验：`us_v7_5_l50_feat_report.json` — val_AUC 0.7452，命中率17.1%（与基线持平）
- L50-Weighted实验：`us_v7_5_l50_weighted_report.json` — 回归式降至14.3%（倒退）
- 后处理过滤：`us_v7_5_filtered_report.json` — 硬过滤5.7%，软过滤8.6%（大幅倒退）
- 前瞻验证8月日志 — 每月的Top5命中率统计均在14-20%之间波动

**根因：** yfinance的OHLCV数据对尾部爆发事件的预测信息有限。布林带+成交量已是最高提取效率的信息路由。

### 2. ⚠️ 月度亏损概率1.9%不保证不连亏

**问题描述：** 前瞻验证期望月收益+16.5%，亏损月概率仅1.9%。但1.9%不是0，**回测540个月中理论上有10个亏损月**。连亏2个月的概率~0.04%（约2500个月发生一次），连亏3个月~0.001%。

**影响范围：** 极低概率但极端效应——连亏可能触发信心崩溃+盲目调参。

**严重程度：** 中等偏低 —— 1.9%单月亏损概率≈每4.4年一次亏损月。完全可以接受。

**分析来源：** 前瞻回测日志中各月的盈亏统计。68个月中1个月亏损（2024-01小亏~-3%），实际亏损率1.47%。

### 3. ⚠️ 布林带宽占主导（单特征风险）

**问题描述：** 特征重要性显示bb_width（布林带宽）贡献12.4%的Gain，是最重要特征的2.4倍。第二名bb_x_vol只有5.3%。模型过度依赖布林带信号。

**影响范围：** 如果某市场阶段布林带信号大面积失效（如横盘窄幅震荡+无放量），模型可能会全面无效。

**严重程度：** 中高 —— 这是一个模型脆弱性输入点。但目前没有实验验证布林带失效场景的真实影响。

**分析来源：** `us_v7_5_l50_report.json` 中 `top15_features` 的Gain分布；`us_v7_5_l50_feat_report.json` 中 `feature_importance_top20`

### 4. ⚠️ yfinance数据更新延迟+速率限制

**问题描述：** yfinance的日K数据更新不一定和美股收盘即时对齐。高峰时段可能延迟5-15分钟。下载~600只股票需3-5分钟，90%时间花在限速等待。

**影响范围：** 盘后评分可能不是真正的"收盘后立即"——和实际收盘有5-20分钟延迟。且yfinance可能会在小盘股上返回缺失数据。

**严重程度：** 低 —— 5-20分钟延迟对T+1交易策略影响极小。如果换minishare us_daily需升级token。

**分析来源：** 生产脚本运行时间日志；`TOOLS.md` 数据源对照表

### 5. ⚠️ 54.5%的收益集中在5只超级爆发

**问题描述：** 8月跨期回测中，54.5%的累计收益来自前5大赢家（排名前1%的命中事件）。去掉这些事件后，模型收益大幅下降。

**影响范围：** 模型本质上是**彩票逻辑**——大部分收益来自少数极端事件。如果连续几周没有爆发，净值会低速增长。

**严重程度：** 中 —— 策略本质如此，不是bug。但用户需要理解这不是稳定的"每月+16.5%"的提款机。

**分析来源：** 前瞻回测日志中的收益分布分析（月度超额收益分解，保存于回测输出）；模型README旧版中的"54.5%集中度"数据

### 6. ⚠️ 买入价假设为次日开盘价（滑点风险）

**问题描述：** 模型假设使用评分次日的开盘价买入。但$1-10的低流动性股的开盘价可能被大单打偏0.5-3%，这直接腐蚀模型收益。回测时无法精确模拟滑点。

**影响范围：** 真实收益可能比回测低10-20%。在流动性较差的三四线小盘中尤其严重。

**严重程度：** 中 —— 执行计划中通过 `us_filtered_syms.json` 过滤日均成交额<$5M的股票来缓解。但仍有0.1-0.5%滑点影响。

**分析来源：** `EXECUTION_PLAN_V75.md` "假设与风险"第1条；回测使用次日开盘价的假设
