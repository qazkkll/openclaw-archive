## 一、起源与演进

> 填写说明：从"为什么要做这个模型"讲起。不要只列技术参数，要写心路历程——试过什么、为什么放弃、为什么选现在这条路。

### 1. 从A1到A2

**A1资金流模型**（2026-06初）设计了三层架构：
- Layer 1: 市场状态分类（防守/中性/激进）
- Layer 2: 数据接入
- Layer 3: 个股评分（XGBoost回归/分类）
- Layer 4: 买卖决策

理想很丰满：三层叠加，资金流 + 技术面 + 状态判断。

### 2. 两次致命打击

**第一次：A1纯资金流独立回测 CAGR -26.94%**

资金流数据（北向/大户/散户）在A股存在严重滞后性——暴涨时大单流出、暴跌时大单流入。用资金流做独立选股策略，T3 CAGR **-26.94%**。从那天起，资金流降级为"实时盘中提示"，不再做独立选股。

分析来源：`scripts/a1_layer3_xgb.py` 中 `backtest_a1_pure_moneyflow()` 函数的回测输出。回测区间2017-2026，日频调仓，资金流Top3买入 → CAGR -26.94%。

**第二次：A1+L3融合 vs 纯L3 无差异**

花了大量时间做A1资金流 + L3回归的融合评分权重优化。回测结果：融合CAGR +62%，纯L3 +60%。**差值的2%在回测噪声范围之内。**

分析来源：`scripts/a1_layer3_xgb.py` 中 `test_fusion()` 函数的两组并行回测。融合权重从0.1到0.9网格搜索，最终结论两种方案的95%置信区间重叠。

结论：**纯L3回归评分就是A股选股的最强信号。A1资金流只做实时信号，不做选股融合。**

### 3. 模型版本对照

| 版本 | 类型 | 特征数 | 目标 | 状态 |
|:---|:---|---:|:---|:---|
| `a1_l3_clf_5d` | 分类 XGBoost | 37 | 5日涨/跌 Binary | ❌ Q5区分度53.7% 弱信号 |
| `a1_layer3_xgb_10d` | **回归 XGBoost** | **37→38** | **10日涨幅回归** | ✅ **主力生产模型** |

**为什么选回归放弃分类：** 分类模型Q5只有53.7%，和抛硬币差别不大。回归模型直接输出10天预期涨跌幅，Top3排序选股区分度远高于分类。

分析来源：`scripts/a1_layer3_xgb.py` 中 `run_clf_training()` 的验证输出——分类模型五分位层差<10%（Q5-Q1），选股信号弱。

### 4. 候选池的演化

| 阶段 | 候选池 | 原因 |
|:---|:---|:---|
| 初始 | 30只（质量池Top30） | 回测起步，先看Top3效果 |
| Walk-Forward | 400只 | 发现更多候选不影响Top3命中率，但能改善夏普 |
| 生产 | **400只**（成交量大、价格≥1元、非ST） | Walk-Forward 7段验证通过 |

分析来源：`scripts/a1_layer3_xgb.py` 中 `walk_forward()` 的参数扫描结果——候选池从30→200→400，夏普从0.85→0.95→1.03，再增大候选池至800夏普不再提升。

## 二、模型设计

> 填写说明：技术细节，特征什么、为什么选这些特征。不要太浅，要让别人看了能复现。

### 1. 模型架构

```
输入（38特征）
  ├── 技术特征（18个）：MA乖离、波动率、RSI、动量、成交量比、相对强度
  └── 资金流特征（20个）：北向/大户/散户净流、累计异常流
        ↓
  XGBoost回归器：树深度6, 学习率0.05, subsample 0.8, colsample 0.8, 500棵树, early_stop 50
        ↓
  输出：未来10日预期涨跌幅（回归得分，未经校准，绝对值不可信，仅用于排序）
        ↓
  按得分排序 → Top3-5买入候选
```

### 2. 特征详解

**技术特征（18个）：**

| 特征 | 含义 | 计算方式 |
|:---|:---|---:|
| `pct_ma5/10/20/60/120` | 价格偏离各均线% | (close/ma-1)*100 |
| `ma20_slope/ma60_slope` | 均线斜率% | 前后21天均值变化百分比 |
| `ma_align` | 均线排列 | 6项布尔得分（短期>长期+价格站上均线） |
| `vol_10d/vol_60d` | 10/60日波动率 | 日均|回报|% |
| `vol_ratio` | 波动率比值 | vol_10d/vol_60d |
| `atr20_pct` | ATR占价格% | 20日平均真实波幅/收盘价 |
| `ret_5d/10d/20d/60d` | 收益率 | 区间百分比 |
| `rsi14` | RSI | 标准14日RSI |
| `vol_ratio_5_20` | 成交量比 | 5日均量/20日均量 |
| `ret20d_pct` | 20日收益百分位 | 过去n个20日窗口中的排名百分位 |

**资金流特征（20个）：**

| 特征 | 含义 | 来源 |
|:---|:---|---:|
| `net_mf_1d/5d/10d/20d/60d` | 当日/累计净流入 | tushare moneyflow |
| `lg_net_1d/5d/10d/20d` | 大单净流入 | 买大单-卖大单 |
| `elg_net_1d` | 超大单净流入 | 超大单净流 |
| `md_net_1d` | 中单净流入 | 中单净流 |
| `major_net_1d/5d/10d/20d/60d` | 主力资金净流入 | (大单+超大单)-(卖大单+卖超大单) |
| `lg_pct` | 大单成交占比% | 大单成交额/总成交额 |
| `elg_pct` | 超大单占比% | 超大单/总成交额 |
| `major_ratio` | 主力净流占比% | (主力买-主力卖)/总主力成交 |

### 3. 为什么是38个特征而不是37个？

早期版本有37个特征（18技术 + 18资金流 + 1个融合权重）。Walk-Forward训练时发现资金流特征包含20个更全（增加了`net_mf_60d`和`major_net_60d`两个长周期累计特征），长期趋势判断更稳。

**特征重要性排名（按XGBoost gain）：**
1. `rsi14` — 超买超卖是最强信号
2. `pct_ma60` — 中长期趋势偏离
3. `vol_ratio_5_20` — 短期放量信号
4. `major_net_5d` — 5日主力资金累计
5. `ret20d_pct` — 相对强弱百分位

> 填写说明：别人拿到这个模型，该怎么办？直接给步骤。

### 2. 选股决策

看完输出后做3件事：
1. **看大盘情绪** — 如果"极冷"或"过热"，持仓减半或暂停新买入

## 六、缺点与局限性

> 填写说明：这是最重要的部分之一。不要只说好话，老实承认模型不行的地方。每条必须写**分析来源**——数据/回测/生产中的哪个输出、哪个日志、哪次验证发现的。这样后面的人能复现你的结论。

### 已知问题

#### 1. ⚠️ 评分偏高（回归输出未校准）

**问题描述：** L3回归模型对RSI<10的极端超卖股票，给出+29%~+35%的10日预期涨幅分。测试集Top1%平均回报只有+21.70%，+35%已超出模型验证范围内的合理区间。

**影响范围：** 影响Top1%推荐，高分股票可能被过度高估。排名顺序基本不变，但分值绝对值不可信。

**严重程度：** 中等偏高 —— 用户可能误以为+30%是实际预期涨幅

**分析来源：**
- 测试集验证输出：`D:/openclaw/data/test_output_layer3_10d_2026-06-12.json` 中 Top1% 的 avg_actual_return = +21.70%
- 生产评分结果：上述显示 Top1 评分+35.62% 远超 +21.70%
- Walk-Forward R²=0.162 —— 模型解释力有限，预测值天然偏大

**根因：**
- 回归损失函数（RMSE）对大偏差样本的惩罚还不够
- 训练数据中极端超卖样本少（RSI<10的样本占比<3%），模型对这类样本拟合过度

**修复方向（搁置中）：**
- 评分后加校准层：`calibrated_score = (score - mean_baseline) / std_baseline * 10.0 + 5.0`
- 或用Platt缩放/Isotonic Regression把输出映射到±20%区间

#### 2. ⚠️ 2023年窄幅震荡行情亏损

**问题描述：** Walk-Forward第4段（验证期2023全年）CAGR -12%，夏普 -0.31。模型在大盘窄幅震荡+主题驱动行情下无法正常工作。

**影响范围：** 每年都可能遇到的行情类型，平均约2-3年一次

**严重程度：** 高 —— 连续亏损会导致信心崩塌和盲目调参

**分析来源：**
- Walk-Forward结果表（见第一节第5小节）：段4 CAGR -12%，唯一亏损段
- 回测日志 `D:/openclaw/data/walk_forward_results_2026-06-12.json`
- 2023年A股特征：全年上证综指振幅<15%，AI主题行情（CPO/大模型/算力）贯穿全年，多数量化模型失效

**应对措施：**
- 2023年的应对比回测后优化：-15%止损 + 持仓上限20只
- 实测：如果加上-15%止损，该段回撤从-32%缩到约-15%
- 后续出现类似行情时，建议降低仓位至50%以下

#### 3. ⚠️ 资金流数据滞后且不稳定

**问题描述：** tushare pro的moneyflow接口更新速度不稳定，有时比K线滞后1-2天。导致评分时部分股票资金流特征为0或旧数据。

**影响范围：** 评分时约8-12%的股票因资金流缺失被跳过（候选池400只中约30-40只）

**严重程度：** 中等

**分析来源：**
- 生产日志：`a1_layer3_5_scoring.py` 运行时输出"无资金流39只"
- 资金流parquet文件 `moneyflow_core.parquet` 的 `trade_date` 分布检查
- 回测时使用inner join对齐K线和资金流，生产环境无法完全复现相同的对齐条件

**应对措施：**
- 自动跳过无资金流股票（脚本已实现）
- 资金流滞后1-2天时，用最近可用数据替代

#### 4. ⚠️ 候选池只有400只

**问题描述：** 当前候选池为流动性前400只。这基本覆盖沪深300+中证500核心成分股，但**不覆盖小盘股、次新股、北交所**。

**影响范围：** 小盘股和次新股无法评分。这类股票波动大、α更高，但也更难预测。

**严重程度：** 低（对于A股模型，400只核心股已经覆盖了90%的机构持仓）

**分析来源：**
- 候选池构建逻辑：`a1_layer3_5_scoring.py` 中 `build_pool()` 函数
- 参数扫描：候选池从30→800步进测试，400之后夏普不再提升

**应对措施：**
- 如果目标是小盘股投资，可以单独拉一个小盘候选池（400-800只），单独训练模型
- 当前400只候选池已覆盖所有流动性好、适合中短线操作的标的

#### 5. ⚠️ 未做评分校准（新建，从生产中发现）

**问题描述：** 详见第1条。回归模型输出未经校准，评分的绝对值不可信，只能做排序参考。

**影响范围：** Top1-5评分极高（+29~+35%），让用户产生不切实际的收益预期。**Andy提出明确不接受。**

**严重程度：** 高

**分析来源：** 2026-06-12生产运行结果，Top5评分+29%~+35%。经Andy确认不能接受。

**修复方向：** 搁置，等待后续处理。

## 附录：模板使用说明

### 这个README的结构

```
1. 模型元数据          ← 快速概览
2. 起源与演进          ← 心路历程，谁都想做，为什么做成这样了
3. 模型设计            ← 特征、架构、为什么选这些
4. 数据源              ← 数据从哪来，有什么坑
5. 评分管线（生产封装） ← 一条命令怎么跑，输出长什么样
6. 缺点与局限性        ← ⭐最重要的部分，诚实面对问题
7. 风险边界            ← 什么时候会死
8. 文件路径            ← 所有相关文件在哪
9. 更新记录            ← 谁改了、改了什么
```

### 每个新模型必须遵守

1. **COPY 此文件作为模板** → 删掉内容，按各节指引填写
2. **"缺点与局限性"必须写** → 每条带上"分析来源"（日志/回测/生产输出），不能只是"可能有问题"
3. **"起源"必须写真实历程** → 试过什么、失败了什么，不能跳过
4. **"模型元数据"必须更新** → version+1，最后更新时间
5. **"文件路径"必须准确** → 路径不对等于不存在
6. **"更新记录"每改必写** → 不要等到全部写完才补

### 常见错误

- ❌ 只写优点不写缺点 → 别人拿了一用就炸
- ❌ 分析来源写"回测发现"但不说哪个文件 → 别人无法复现
- ❌ 说"这是一个XGBoost模型"就完了 → 特征呢？参数呢？
- ❌ 路径写相对路径但不知道相对谁 → 必须写绝对路径或标准项目根
