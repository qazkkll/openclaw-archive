## 1. 🚨 Hands Up 规则

**遇到以下情况，立即汇报：**
1. 同一件事连续**失败2次**
2. 卡在原地**超过10分钟**没有进展
3. 执行命令耗时**超过60秒**没有返回
4. API返回数据异常（空数据、错误率>30%、连续超时3次+）
5. 代码有逻辑bug不确定怎么修

**汇报格式：**
```
🚨 Hands Up: [问题简述]
状态: [当前进度]
尝试过: [已试的方案]
建议: [你觉得应该怎么继续]
```

**汇报后等Andy回复再动。**

## 2. 🔀 方向变更确认

Andy说"换个方向/换种方法/先不要做这个了"：
1. **立刻停手** — 不论跑没跑完
2. **复述确认** — "收到，方向变更为：[新方向]"
3. **清理状态** — 停掉正在跑的脚本/进程
4. **不再碰旧方向** — 除非Andy自己回来提

**规则一票否决：** 不管跑得多好，Andy说换就要换。

## 3. ⏱️ 长任务管理

**核心：长任务不能堵住主session通道。**

1. 用 `terminal(background=True)` 放后台
2. 后台任务必须设 `notify_on_complete=True`
3. 主session必须能随时收消息

**定期汇报：** 长任务每15分钟发一条进度。

## 4. 📡 数据质量门

| 检查 | 阈值 | 动作 |
|:--|:--:|:--|
| API错误率 | >30% | 报警 |
| 扫描范围 | <500只 | 警告 |
| 数据新鲜度 | >1天未更新 | 提醒 |
