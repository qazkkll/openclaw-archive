## 1. 🚨 Hands Up 规则

**当遇到以下情况，立即停下手头的事，汇报给Andy：**
1. 同一件事连续**失败2次** — 换个方法试了两次还是不行，停，汇报
2. 卡在原地**超过10分钟**没有进展 — 不再出数据、不再有新输出
3. 有一条执行命令耗时**超过60秒**没有返回 — 不管是回测还是文件读写
4. API返回数据异常（空数据、错误率>30%、连续超时3次+）
5. 代码有逻辑bug不确定怎么修

**怎么汇报：**
```
🚨 Hands Up: [问题简述]
状态: [当前进度]
尝试过: [已试的方案]
建议: [你觉得应该怎么继续]
```

**汇报后等Andy回复再动。** 不等就继续白费功夫。

## 2. 🔀 方向变更确认

Andy任何时候说"换个方向/换种方法/先不要做这个了"：
1. **立刻停手** — 当前任务不论跑没跑完
2. **复述确认** — "收到，方向变更为：[新方向]"
3. **清理状态** — 如果有正在跑的脚本/进程，先停掉
4. **不再碰旧方向** — 除非Andy自己回来提

**规则一票否决：** 不管跑得多好、多接近出结果，Andy说换就要换。

## 3. ⏱️ 长任务 Session 管理

**核心问题：回测/扫描不能堵住主session通道。**

做长任务时：
1. **用 `exec(background=True)` 放后台** — 不要用主session跑同步慢操作
   ```
   exec(command='python3 scripts/xxx.py', background=True, yieldMs=5000)
   ```
2. **用 `sessions_spawn` 派子任务** — 独立session，不堵主通道
3. **任何时候主session必须能收消息** — 如果发现堵住，杀进程也要腾出通道

**定期汇报制度：**
- 长任务每**15分钟**发一条进度（哪怕只是"还在跑"）
- 无论Andy有没有问
